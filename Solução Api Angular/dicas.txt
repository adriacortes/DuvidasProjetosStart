1. Sempre, sempre que tiver um erro analise seu código por partes!
O problema é na API ou no FRONT?
Neste caso, você pode utilizar um programa como o POSTMAN para analisar a parte
da API e resolver qualquer problema encontrado nela, para assim prosseguir para a parte
do FRONT(angular) sabendo que sua API esta ok.

Já peguei vários casos onde o problema inicial era na API, de nada adiante
tentar corrigir o FRONT se o erro esta na raiz.

Então sempre vamos por partes, ok!

2. Você pode usar o debugger para analisar os dados que chegam pelo
CONSOLE em X método, assim ficara mais fácil analisar o possível problema.

Exemplo:

obterCursos():Observable<Curso[]>{   
    return this.http.get(this.url+"listar")   
    .pipe( map((res:any) =>{    
       this.vetor = res; 
       debugger //com isso,vc pode ver os dados vindo do seu banco de dados via console !
       return this.vetor;
      }))
}
